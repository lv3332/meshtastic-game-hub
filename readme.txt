Meshtastic Game Hub (Offline Edition) Це автономна ігрова платформа для Meshtastic-мереж. 
Вона дозволяє грати в ігри (Морський бій, Шахи, Шашки) прямо в браузері, передаючи ходи 
через радіоефір без інтернету.

1. Архітектура. Система складається з двох частин:bridge.py: Python-сервер (міст). 
Він з'єднує ваш Meshtastic-модем із браузером через WebSocket та роздає ігрові файли 
по HTTP.*.html: Ігрові клієнти. Вся логіка (правила ігор) працює локально в браузері, 
через радіо передаються лише короткі текстові команди.

2. Підготовка та встановлення. Вимоги:Python 3.10+ (рекомендовано 3.14). Meshtastic-модем, 
підключений по USB.Встановлення залежностей: Створіть папку проєкту і покладіть туди всі файли 
(bridge.py, ігрові .html файли, папку js, css, img).Встановіть необхідні бібліотеки:
 Для macOS / Linux:
python3 -m pip install meshtastic websockets pypubsub

3. Як користуватися. Запуск сервера: Підключіть модем по USB. Запустіть міст: python3 bridge.py
Сервер видасть адресу: http://127.0.0.1:8890. Гра в браузері: Відкрийте браузер і перейдіть за 
адресою http://127.0.0.1:8890.Натисніть кнопку "Підключити". Коли з'явиться зелене повідомлення — 
модем готовий.Введіть короткий нік (Short Name) суперника (його можна побачити в налаштуваннях модема 
суперника). Якщо залишити поле пустим — повідомлення піде в загальний канал (Broadcast).
Розставте кораблі / натисніть "Готовий до гри".Після того, як суперник теж натисне кнопку, дошка 
активується.

4. Troubleshooting (Поради) Проблема Причина Рішення OS Error: could not bind Порт 8890 зайнятий 
Переконайтеся, що інший процес не займає порт 8890.Modem not found Модем не підключений 
Перевірте USB-кабель. Спробуйте інший порт. Немає з'єднання WebSocket відпав Перезапустіть bridge.py 
та оновіть сторінку в браузері.Фігури не рухаються Суперник не отримав пакет Перевірте, чи правильно 
вказано нікнейм або чи не "висить" Mesh-мережа.

5. Додавання нових ігорЩоб додати нову гру, просто створіть файл new_game.html.Логіка: використовуйте 
префікс NEWGAME: для ваших команд.Маршрутизація: сервер bridge.py автоматично розпізнає новий файл і 
віддасть його браузеру за адресою http://127.0.0.1:8890/new_game.html.Фільтрація: Обов'язково додайте 
префікс нової гри у кортеж if text.startswith(("MB:", "CHESS:", "NEWGAME:")): у файлі bridge.py.Порада 
для поля: Якщо ви використовуєте Orange Pi або Raspberry Pi, налаштуйте запуск bridge.py через systemd, 
щоб ігровий сервер підіймався автоматично відразу після увімкнення живлення живлення (Powerbank).
Тепер ти можеш не просто грати, а бути "хостом" для цілої радіо-ігротеки! 🎮📻
