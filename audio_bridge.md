## 🔊 Акустичний лінк (audio_bridge.py)

Цей режим перетворює ваш пристрій на цифровий TNC-модем, який передає ігрові пакети та повідомлення чату через звук (Data-Over-Sound). Це дозволяє використовувати будь-які аналогові радіостанції для передачі цифрових даних без необхідності підключати їх кабелем.

### Як це працює
1. Запустіть сервер: `python3 audio_bridge.py`.
2. Надайте терміналу дозвіл на використання мікрофона (система може запитати про це при першому запуску).
3. Відкрийте браузер за адресою `http://127.0.0.1:8890`.
4. **Для гри через ефір:** Увімкніть режим **VOX** на вашій рації, покладіть її поруч із комп'ютером/смартфоном. Сервер автоматично генеруватиме тон для відкриття каналу перед кожною передачею.

### Основні налаштування (редагуються у файлі)
* `GGWAVE_PROTOCOL = 1`: Оптимальний режим для голосового радіоефіру. Якщо ви хочете грати ноутбуками в одній кімнаті без рацій, змініть на `4` (ультразвук), щоб передача була безшумною для людей.
* `VOX_DELAY_MS = 600`: Час "прогріву" ефіру (в мілісекундах). Якщо ваша рація "ковтає" початок повідомлення і воно не розшифровується — збільште цей параметр до `800` або `1000`.

## 📱 Встановлення та запуск на Android (через Termux)

Ви можете розгорнути цей ігровий хаб на будь-якому Android-смартфоні, перетворивши його на портативний автономний сервер.

### 1. Підготовка середовища
1. Встановіть актуальну версію **Termux** (рекомендується завантажувати з F-Droid).
2. Запустіть Termux і оновіть системні пакети:
   \`\`\`bash
   pkg update && pkg upgrade -y
   \`\`\`
3. Надайте доступ до файлової системи:
   \`\`\`bash
   termux-setup-storage
   \`\`\`
4. **Важливо:** Зайдіть у налаштування Android ➔ Додатки ➔ Termux ➔ Дозволи, та **вручну надайте доступ до Мікрофона**.

### 2. Встановлення системних залежностей
Для коректної роботи аудіо-модулів та їх компіляції під ARM-архітектуру смартфона, встановіть компілятори та аудіо-бібліотеки:
\`\`\`bash
pkg install python portaudio clang make binutils -y
\`\`\`

### 3. Встановлення Python-бібліотек
Завантажте необхідні пакети через `pip`. Компіляція `pyaudio` та `ggwave` може зайняти кілька хвилин — не закривайте термінал:
\`\`\`bash
pip install pyaudio ggwave websockets
\`\`\`
*(Примітка: якщо ви плануєте використовувати стандартний USB/IP міст `bridge.py`, додатково встановіть `pip install meshtastic pypubsub`).*

### 4. Запуск хабу
Перейдіть у папку, де збережені ваші `.html` та `.py` файли (наприклад, у внутрішню пам'ять смартфона) і запустіть міст:
\`\`\`bash
python audio_bridge.py
\`\`\`
Згорніть Termux (залишивши його працювати у фоні), відкрийте мобільний браузер (Chrome/Firefox) і перейдіть на `http://127.0.0.1:8890`. Ваш портативний радіо-ігровий центр готовий!
